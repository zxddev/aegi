# AEGI 高级分析能力调研报告

> 作者：白泽
> 日期：2026-02-11
> 目标：找出能让 AEGI 从"情报搜集工具"升级为"专家级参谋助手"的开源项目和算法

---

## 一、核心差距分析

AEGI 要做的不是搜索引擎，而是**分析引擎**。差距在于：

| 能力 | 当前状态 | 目标状态 |
|------|----------|----------|
| 因果推断 | 简单的因果关系提取 | 反事实推理、因果链发现、干预效果预测 |
| 事件预判 | 无 | 基于历史模式预测未来事件概率 |
| 信息鉴别 | 来源可信度评分 | 多源交叉验证、虚假信息检测、宣传识别 |
| 假设分析 | 基础 ACH | 贝叶斯 ACH、证据权重动态更新、对抗性评估 |
| 知识图谱推理 | 存储 + 查询 | 链接预测、时序推理、隐含关系发现 |
| 态势感知 | 无 | 碎片信息拼图、趋势识别、转折点检测 |

## 二、七大方向调研结果

### 1. 因果推断与因果发现

| 项目 | GitHub | 成熟度 | 集成难度 | 推荐度 |
|------|--------|--------|----------|--------|
| **DoWhy** (Microsoft) | github.com/py-why/dowhy | ⭐⭐⭐⭐ 生产可用 | 低，纯 Python | 🔴 必须有 |
| **CausalNex** (QuantumBlack) | github.com/quantumblacklabs/causalnex | ⭐⭐⭐ 成熟 | 低，基于 networkx | 🟡 推荐 |
| **causal-learn** (CMU) | github.com/cmu-phil/causal-learn | ⭐⭐⭐ 学术级 | 中 | 🟡 推荐 |
| **gCastle** (华为) | github.com/huawei-noah/trustworthyAI/tree/master/gcastle | ⭐⭐⭐ | 中 | 🟢 备选 |

**集成建议：**
- **DoWhy 必须集成**。它提供完整的因果推断流程：建模 → 识别 → 估计 → 反驳。可以直接用在 AEGI 的因果推理模块中。
- 用 CausalNex 做因果图的自动发现（从数据中学习因果结构），与现有 Neo4j 图谱结合。
- LLM + DoWhy 的组合：用 LLM 提出因果假设，用 DoWhy 做统计验证。

### 2. 事件预测与预判

| 项目/方法 | 来源 | 成熟度 | 集成难度 | 推荐度 |
|-----------|------|--------|----------|--------|
| **GDELT Project** | gdeltproject.org | ⭐⭐⭐⭐ 数据源 | 低，REST API | 🔴 必须有 |
| **ICEWS** | dataverse.harvard.edu | ⭐⭐⭐⭐ 数据源 | 中 | 🟡 推荐 |
| **CAMEO 编码体系** | eventdata.parusanalytics.com | ⭐⭐⭐⭐ 标准 | 低 | 🔴 必须有 |
| **Temporal Point Process** (NHP/THP) | 学术论文 | ⭐⭐ 研究级 | 高 | 🟢 备选 |
| **超级预测者方法论** | Tetlock 研究 | 概念级 | 中 | 🟡 推荐 |

**集成建议：**
- **GDELT 必须接入**。它是全球最大的开放事件数据库，实时更新，覆盖全球媒体。AEGI 的 OSINT 采集应该把 GDELT 作为核心数据源之一。
- **CAMEO 编码体系**用于事件分类标准化（谁对谁做了什么），是情报分析领域的通用语言。
- 事件预测的实用方案：不用复杂的时序模型，用**历史模式匹配 + LLM 推理**。从 GDELT/ICEWS 历史数据中找相似情境，让 LLM 基于历史模式做预判。
- 超级预测者方法论可以算法化：分解问题、基准概率、增量更新、对立观点。这个可以用 LLM + structured output 实现。

### 3. 信息真伪鉴别

| 项目 | GitHub | 成熟度 | 集成难度 | 推荐度 |
|------|--------|--------|----------|--------|
| **ClaimBuster** | github.com/idir-lab/ClaimBuster | ⭐⭐⭐ | 中 | 🟡 推荐 |
| **LIAR dataset + models** | 学术 | ⭐⭐⭐ 数据集 | 中 | 🟢 备选 |
| **Factiverse AI** | factiverse.ai | ⭐⭐⭐ | 中，API | 🟡 推荐 |
| **多源交叉验证** | 自建 | — | 中 | 🔴 必须有 |

**集成建议：**
- 多源交叉验证是最实用的：同一事件从不同来源采集，自动比对一致性和矛盾点。这个 AEGI 已有基础（多源 OSINT + 声明提取），需要加一层**自动对比逻辑**。
- ClaimBuster 可以用来识别"值得验证的声明"，作为 claim_extractor 的增强。
- LLM 在 fact-checking 上已经很强，关键是给它足够的上下文（多源证据）和结构化的验证框架。

### 4. 高级分析方法（ACH 增强、SATs）

| 方法 | 来源 | 成熟度 | 集成难度 | 推荐度 |
|------|------|--------|----------|--------|
| **贝叶斯 ACH** | 学术论文 | 概念级 | 中 | 🔴 必须有 |
| **Argument Mining (论证挖掘)** | 多个开源项目 | ⭐⭐⭐ | 中 | 🟡 推荐 |
| **认知偏见检测** | 自建 + LLM | — | 低 | 🔴 必须有 |
| **IARPA ACE 方法论** | 公开论文 | 概念级 | 中 | 🟡 推荐 |

**集成建议：**
- **贝叶斯 ACH 是最大的升级点**。现有 ACH 是静态的（一次性评估），贝叶斯 ACH 支持：新证据到来时动态更新假设概率、量化证据权重、计算信息增益。这个可以用 Python 的 `pgmpy`（贝叶斯网络库）实现。
- 认知偏见检测：用 LLM 检查分析过程中的常见偏见（确认偏见、锚定效应、镜像思维等）。这个 AEGI 已有 `bias_detector.py`，需要增强。
- 论证挖掘：从文本中自动提取论点、论据、论证结构。可以增强 claim_extractor 的能力。

### 5. 知识图谱推理

| 项目 | GitHub | 成熟度 | 集成难度 | 推荐度 |
|------|--------|--------|----------|--------|
| **PyKEEN** | github.com/pykeen/pykeen | ⭐⭐⭐⭐ 生产可用 | 低，纯 Python | 🔴 必须有 |
| **DGL-KE** | github.com/awslabs/dgl-ke | ⭐⭐⭐ | 中 | 🟢 备选 |
| **PyG (PyTorch Geometric)** | github.com/pyg-team/pytorch_geometric | ⭐⭐⭐⭐ | 高 | 🟡 推荐 |
| **AmpliGraph** | github.com/Accenture/AmpliGraph | ⭐⭐⭐ | 低 | 🟢 备选 |

**集成建议：**
- **PyKEEN 必须集成**。它支持 TransE、RotatE、CompGCN 等主流 KG embedding 模型，可以做：
  - **链接预测**：预测实体之间可能存在但尚未发现的关系（"隐含关系发现"）
  - **实体分类**：基于图结构推断实体属性
  - **异常检测**：发现不符合图谱模式的异常关系
- 与现有 Neo4j 图谱结合：从 Neo4j 导出子图 → PyKEEN 训练 → 预测结果写回 Neo4j。
- 时序 KG 推理（预测关系随时间的变化）可以后期加。

### 6. 垂直领域模型与数据集

| 资源 | 类型 | 说明 | 推荐度 |
|------|------|------|--------|
| **GDELT GKG** | 数据源 | 全球知识图谱，实体/事件/主题/情感 | 🔴 必须有 |
| **CAMEO** | 编码标准 | 冲突与调解事件编码 | 🔴 必须有 |
| **Wikidata** | 知识库 | 实体消歧的权威参考 | 🟡 推荐 |
| **MITRE ATT&CK** | 框架 | 网络威胁情报分析框架 | 🟢 领域特定 |
| **ACLED** | 数据源 | 武装冲突与抗议事件数据 | 🟡 推荐 |

**集成建议：**
- GDELT + CAMEO 是情报分析的基础设施，必须接入。
- Wikidata 可以增强实体消歧（现有别名表太小，Wikidata 有完整的多语言别名）。
- 不需要专门的垂直领域预训练模型——现有的通用 LLM（GPT-4/Claude）在情报分析任务上已经很强，关键是给它正确的上下文和分析框架。

### 7. 端到端情报分析系统

| 项目 | 来源 | 说明 | 参考价值 |
|------|------|------|----------|
| **OpenCTI** | github.com/OpenCTI-Platform | 开源网络威胁情报平台 | ⭐⭐⭐⭐ 架构参考 |

## 三、综合推荐：Top 5 必须集成

如果只能选 5 个，选这些：

| 优先级 | 项目/方法 | 用在哪里 | 为什么 |
|--------|-----------|----------|--------|
| 1 | **DoWhy** | 因果推理模块 | 反事实推理、因果验证，让分析从"关联"升级到"因果" |
| 2 | **PyKEEN** | 知识图谱推理 | 链接预测、隐含关系发现，让 KG 从"存储"升级到"推理" |
| 3 | **GDELT + CAMEO** | OSINT 数据源 + 事件编码 | 全球事件数据的标准化采集，是主动监测的基础 |
| 4 | **贝叶斯 ACH (pgmpy)** | 假设分析引擎 | 动态更新假设概率，量化证据权重，是分析深度的关键 |
| 5 | **超级预测者方法论** | 事件预判模块 | 结构化预测框架 + LLM，让预判有方法论支撑 |

## 四、集成路线图建议

### Phase 1（短期，2-3 周）
- 接入 GDELT 数据源，用 CAMEO 编码标准化事件
- 增强 claim_extractor，加入多源交叉验证逻辑
- 升级 ACH 为贝叶斯 ACH（pgmpy）

### Phase 2（中期，4-6 周）
- 集成 DoWhy 因果推断框架
- 集成 PyKEEN 做 KG 链接预测
- 实现超级预测者方法论的算法化

### Phase 3（长期）
- 时序知识图谱推理
- 论证挖掘增强
- 认知偏见检测增强
- 态势感知仪表盘
