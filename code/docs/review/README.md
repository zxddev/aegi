# AEGI 项目审查文档

## 目录结构

```
docs/review/
├── README.md              ← 本文件，审查索引
└── YYYY-MM-DD/            ← 按日期归档
    ├── 01-proactive-push.md       ← 主动推送机制
    ├── 02-pipeline.md             ← Pipeline 编排
    ├── 03-mcp-gateway.md          ← MCP Gateway 协作
    ├── 04-data-model-kg.md        ← 数据模型与知识图谱
    └── summary.md                 ← 白泽整体评审总结
```

## 规范

- 每轮审查以日期为目录，避免文件堆积
- 文件名前缀编号，保持阅读顺序
- `summary.md` 由白泽在所有审查完成后撰写
- 后续新增审查主题按编号递增（05-xxx.md, 06-xxx.md...）
- 历史审查不删除，作为项目演进记录
